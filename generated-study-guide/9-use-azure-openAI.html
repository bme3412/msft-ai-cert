🤖 Use Azure OpenAI in Foundry Models

Comprehensive guide for provisioning and using Azure OpenAI services

1️⃣
Provision Azure OpenAI

What It Means

🏗️ Foundry Models Resource

You must first create a Foundry Models resource in Azure AI Foundry. This provides access to Azure-hosted OpenAI foundation models (GPT, embeddings, DALL·E).

Provisioning Steps

  • Azure Portal: Create Azure AI Foundry resource
  • Hub/Project Setup: Within hub/project → provision OpenAI model resource
  • Resource Configuration: Set up proper access and permissions
  • Endpoint Configuration: Configure API endpoints and authentication

Quota Assignment

📊 Required Quotas

  • GPT-4 / GPT-4 Turbo: High-performance text generation
  • GPT-3.5: Cost-effective text generation
  • Embeddings: Vector representations for search
  • DALL·E: Text-to-image generation
  • GPT-4V (Vision): Multimodal image + text processing

💡 Provisioning Best Practices

  • Regional Availability: Check model availability by region (not all SKUs are in all regions)
  • Quota Planning: Request quota in advance — GPT-4 may need approval
  • Cost Tracking: Use tags to track cost (model: GPT-4 vs GPT-3.5)
  • Resource Naming: Use consistent naming conventions for easy management

2️⃣
Select & Deploy Models

Available Models

🚀 GPT-4 Turbo

  • Best For: Complex reasoning, chat apps, summarization
  • Advantages: Lower latency, cheaper, extended context windows
  • Use Cases: Production applications, complex multi-turn conversations

⚡ GPT-3.5

  • Best For: Cost-efficient for lightweight tasks (FAQ bots, drafting)
  • Advantages: Useful for high-volume, non-critical use cases
  • Use Cases: Simple Q&A, content generation, prototyping

🎨 DALL·E

  • Best For: Generates images from text prompts
  • Features: Supports image inpainting & editing
  • Use Cases: Marketing visuals, product mockups, creative content

🖼️ Multimodal GPT-4 (Vision)

  • Best For: Accepts image + text input
  • Output: Text only (not images)
  • Use Cases: Interpret charts, caption images, analyze diagrams

💡 Model Selection Best Practices

  • Reasoning/Summarization: → GPT-4 Turbo
  • Cost-sensitive Applications: → GPT-3.5
  • Visual Creative Generation: → DALL·E
  • Image Understanding: → GPT-4 Vision

3️⃣
Submit Prompts

How To Submit

🎮 Azure AI Studio Playground

  • Purpose: GUI for experimenting with prompts
  • Features: Visual prompt testing, parameter tuning
  • Best For: Prototyping, testing, experimentation

💻 SDKs

  • openai: GPT models, embeddings
  • azure-ai-inference: Broader Azure AI Foundry SDK
  • Best For: Production applications, programmatic access

What You Can Generate

  • Natural Language Responses: Chatbot answers, conversational AI
  • Summaries: Meeting notes, transcripts, document summaries
  • Code Completions: Dev copilots, bug fixes, code generation
  • Images (DALL·E): Marketing visuals, product mockups, creative content

Example Implementation

from openai import AzureOpenAI client = AzureOpenAI( api_key="YOUR_KEY", api_version="2024-06-01-preview", azure_endpoint="https://YOUR_RESOURCE.openai.azure.com/" ) response = client.chat.completions.create( model="gpt-4-turbo", messages=[{"role":"user","content":"Summarize this meeting transcript..."}], temperature=0.7, max_tokens=500 ) print(response.choices[0].message["content"])

4️⃣
Build Assistants (Azure OpenAI Assistants API)

What It Is

🤖 Assistants API

A higher-level orchestration API for managing multi-turn AI interactions with advanced capabilities.

Key Features

  • Multi-turn Conversations: Maintain state across sessions
  • Tool Calling: LLM can call external APIs when needed
  • Memory Storage: Persistent context (e.g., "Remember my name is Alex")
  • File Support: Attach docs/images for model grounding

Use Cases

  • Customer Service Assistant: Remembers prior interactions
  • Developer Copilot: Calls code APIs for completions
  • Analyst Assistant: Retrieves company data via API calls
  • Personal Assistant: Manages tasks and reminders

💡 Assistant Best Practices

  • Content Safety: Always integrate content safety filters before returning assistant responses
  • State Management: Store assistant state in databases for continuity across sessions
  • Security: Limit tool access with RBAC + security checks
  • Error Handling: Implement robust error handling for tool calls

🎯 Exam Trick Questions & Common Pitfalls

❓ Question: What's the first step to use Azure OpenAI in AI Foundry?

Answer: Create a Foundry Models resource
Trick: Don't choose individual model deployment - you need the Foundry Models resource first to access all OpenAI models.

❓ Question: Which model is best for complex reasoning and multi-turn chatbots?

Answer: GPT-4 Turbo
Trick: Don't choose GPT-3.5 for complex tasks - GPT-4 Turbo is specifically optimized for production use with better reasoning capabilities.

❓ Question: What's the best model for a low-cost FAQ assistant?

Answer: GPT-3.5
Trick: Don't choose GPT-4 for simple tasks - GPT-3.5 is much more cost-effective for high-volume, simple Q&A applications.

❓ Question: Which model should you use to generate marketing visuals?

Answer: DALL·E
Trick: Don't choose GPT-4 Vision - that's for image analysis, not generation. DALL·E is specifically for text-to-image generation.

❓ Question: What's the best way to test prompts before integration?

Answer: Azure AI Studio Playground
Trick: Don't choose SDKs for testing - the Playground provides a visual interface for prompt experimentation and tuning.

❓ Question: Which API provides multi-turn conversations and tool calling?

Answer: Assistants API
Trick: Don't choose the basic chat completions API - the Assistants API provides higher-level orchestration with memory and tool calling.

❓ Question: What's the key difference between GPT-4 Vision and DALL·E?

Answer: GPT-4V analyzes images and outputs text, DALL·E generates images from text
Trick: Don't confuse their purposes - GPT-4V is for understanding images, DALL·E is for creating images.

❓ Question: Which SDK should you use for broader Azure AI Foundry integration?

Answer: azure-ai-inference
Trick: Don't choose just the openai SDK - azure-ai-inference provides broader integration with the entire Azure AI Foundry ecosystem.
