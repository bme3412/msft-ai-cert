Azure OpenAI Service

Core Capabilities

  • GPT-4, GPT-4 Turbo: Advanced reasoning, multi-turn chat, summarization, content creation
  • GPT-3.5: Lower-cost option for simpler tasks
  • Embeddings API: Convert text into vectors for semantic search, clustering, recommendations
  • DALL¬∑E: Text-to-image generation with inpainting/editing capabilities
  • Azure AI Studio Integration: Prompt testing, fine-tuning, safety filters

Best-fit Use Cases

  • Chatbots & Copilots: Customer service bots, sales assistants
  • Content Generation: Blog drafts, marketing copy, idea brainstorming
  • Summarization: Financial reports, medical transcripts, call center logs
  • Semantic Search & RAG: Embeddings + Azure AI Search integration

Key Considerations

  • Quota & model availability varies by region
  • Cost: Token-based billing (input + output tokens)
  • Security: Azure isolates customer data; enterprise agreements guarantee no training on your data

Azure AI Vision

Core Capabilities

  • Image Analysis 4.0 API: Object detection, image tagging, scene recognition
  • OCR (Read API): Extract printed and handwritten text from documents
  • Spatial Analysis: Count people, track movement in video streams (retail/transport)
  • Face API: Detect face attributes, compare faces, verify identity

Best-fit Use Cases

  • Retail: Footfall analytics, heatmaps
  • Healthcare: Extract text from forms
  • Manufacturing: Detect defects in images
  • Legal/Finance: Digitize documents with OCR

Key Considerations

  • Accuracy depends on image quality and language coverage
  • Security: GDPR restrictions around facial recognition in some regions

  2. Detect objects in a sample image (cars, animals, etc.)
  3. Enable spatial analysis on a video feed

Azure AI Search (Cognitive Search)

Core Capabilities

  • Indexing: Structured + unstructured data (PDF, Office docs, websites)
  • Enrichment Pipelines: Integrate AI Skills (OCR, NER, translation)
  • Semantic Search: Powered by embeddings + vector similarity
  • RAG Integration: Combine with OpenAI for context-aware chatbots

Best-fit Use Cases

  • Enterprise Knowledge Bases: Internal search portals
  • Customer Self-Service: Product manuals, FAQs
  • Legal Discovery: Search across millions of documents

Key Considerations

  • Cost driven by number of indexes & documents
  • Performance: Scale with replicas/shards

üéØ Exam Trick Questions & Common Pitfalls

‚ùì Question: Which service should you use for extracting structured data from invoices?

Answer: Document Intelligence (Form Recognizer)
Trick: Don't confuse with Azure AI Vision OCR - Document Intelligence is specifically designed for structured data extraction from business documents.

‚ùì Question: What's the difference between Azure AI Language and Azure OpenAI for text analysis?

Answer: Azure AI Language is for pre-built text analysis (sentiment, NER, key phrases), while Azure OpenAI is for generative AI tasks (chat, content creation, summarization).
Trick: They're complementary - use Language for analysis, OpenAI for generation.

‚ùì Question: Which service provides the best integration for RAG (Retrieval Augmented Generation)?

Answer: Azure AI Search + Azure OpenAI
Trick: RAG requires both semantic search (AI Search) and language generation (OpenAI). Don't pick just one service.

‚ùì Question: For real-time audio translation in a global meeting, which service?

Answer: Azure AI Speech (Speech Translation)
Trick: This is NOT Azure AI Language translation - Speech handles audio-to-audio translation, Language handles text-to-text.

‚ùì Question: Which service can be deployed via containers for offline use?

Answer: Azure AI Speech, Document Intelligence, and some AI Language features
Trick: Azure OpenAI cannot be containerized - it's only available as a cloud service.

‚ùì Question: What's the billing model for Azure AI Search?

Answer: Per search unit (not per document or per query)
Trick: Search units are based on storage + query volume, not individual searches.

‚ùì Question: Which service requires the most training data for custom models?

Answer: Document Intelligence (5-10 minimum), followed by Speech custom models
Trick: Azure OpenAI fine-tuning needs hundreds of examples, but Document Intelligence can work with fewer samples.

‚ùì Question: For GDPR compliance with facial recognition, what should you consider?

Answer: Azure AI Vision Face API has regional restrictions and requires special approval
Trick: Face API is not available in all regions and may require additional compliance steps.
