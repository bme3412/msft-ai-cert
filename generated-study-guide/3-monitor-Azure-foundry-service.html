Account Keys Management

Key Concepts

Every Cognitive Service/OpenAI resource provides:

  • Primary Key: Main authentication key for API access
  • Secondary Key: Backup key for key rotation
  • Key Rotation: Both keys are valid ‚Üí rotate periodically for security
  • Regeneration: Keys can be regenerated without service interruption

üîê Security Best Practices

  • Azure Key Vault: Store keys in Key Vault ‚Üí apps retrieve at runtime
  • No Hard-coding: Never hard-code keys in source code or config files
  • Regular Rotation: Rotate keys (e.g., every 90 days)
  • Access Control: Limit key access to authorized applications only

üéØ Hands-on Practice

  1. Store API key in Azure Key Vault
  2. Configure an app to retrieve key via Managed Identity
  3. Rotate a key and confirm the app updates without downtime

Authentication Options

API Keys

  • Simple: Easy to implement and use
  • Less Secure: Keys can be compromised or leaked
  • Use Case: Only for quick POCs and development
  • Management: Requires manual key rotation and storage

Managed Identity (Recommended)

  • Azure AD Identity: Assign an Azure AD identity to your app/service
  • No Key Storage: Eliminates need for keys
  • RBAC Integration: Integrates with Role-Based Access Control
  • Automatic Rotation: Microsoft manages credential rotation

Benefits of Managed Identity

  • No Key Management: No key storage/rotation burden
  • Least Privilege: Access via RBAC roles
  • Compliance: Stronger compliance posture (SOC, GDPR, HIPAA)
  • Security: Reduces attack surface and credential exposure

  1. Enable system-assigned managed identity for an Azure Function
  2. Grant access to an AI resource (Cognitive Services Contributor role)
  3. Call the AI endpoint without embedding an API key

üéØ Exam Trick Questions & Common Pitfalls

‚ùì Question: Which tool should you use to debug why an AI application is responding slowly?

Answer: Application Insights
Trick: Don't choose Azure Monitor for application-level debugging - App Insights provides deep telemetry and dependency tracking for AI workloads.

‚ùì Question: What's the most secure way to authenticate an Azure Function to an AI service?

Answer: Managed Identity
Trick: Don't choose API keys - Managed Identity eliminates key management and provides better security with RBAC integration.

‚ùì Question: How should you store API keys for production applications?

Answer: Azure Key Vault
Trick: Never store keys in configuration files or environment variables - Key Vault provides secure, centralized key management.

‚ùì Question: Which metric is most important for tracking OpenAI model costs?

Answer: Token consumption (input + output tokens)
Trick: Don't focus on request count - OpenAI billing is based on token usage, not the number of requests.

‚ùì Question: What's the best way to prevent runaway costs from an AI service?

Answer: Set usage quotas and budget alerts
Trick: Don't rely only on monitoring - proactive cost controls like quotas and budgets prevent unexpected charges.

‚ùì Question: Which tool provides the most detailed log analysis for troubleshooting?

Answer: Diagnostics Logs with KQL queries
Trick: Don't choose basic monitoring tools - Diagnostics Logs with KQL provide the deepest analysis capabilities.

‚ùì Question: How often should you rotate API keys for security?

Answer: Every 90 days (or as per security policy)
Trick: Regular key rotation is essential for security - don't leave keys unchanged indefinitely.

‚ùì Question: What's the difference between primary and secondary API keys?

Answer: Both are valid - secondary is for key rotation without downtime
Trick: Don't think one is for read-only access - both keys have the same permissions and are used for seamless rotation.
