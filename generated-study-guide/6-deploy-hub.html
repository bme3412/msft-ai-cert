1. Hub (Top-Level Workspace)

Concept

üè¢ Hub Definition

A Hub is the top-level organizational container in Azure AI Foundry. Think of it like a resource group dedicated to AI projects.

Key Points

  • Multi-Project Container: Each hub can contain multiple projects
  • Shared Infrastructure: Manages authentication, monitoring, governance
  • Centralized Dashboards: Provides cost, usage, and compliance visibility
  • RBAC Management: Apply roles and permissions at hub level

üí° Hub Best Practices

  • Business Alignment: Create hubs aligned with business units (e.g., Finance AI Hub, HR AI Hub)
  • RBAC Roles: Apply roles at hub level (e.g., data scientist, contributor, reader)
  • Cost Tracking: Use tags to track costs across environments (dev/test/prod)
  • Governance: Establish consistent policies and standards across all projects

üéØ Hands-on Practice

  1. In Azure AI Studio, create a new hub called "Enterprise-AI-Hub"
  2. Configure RBAC roles for different team members
  3. Set up cost tracking tags for environment separation

2. Project (Scoped Environment)

Concept

üìÅ Project Definition

A Project is a workspace inside a hub, scoped to a specific solution or app (e.g., Customer Support Chatbot Project).

Project Contents

  • Datasets: PDFs, images, structured data
  • Models: GPT-4, embeddings, fine-tuned versions
  • Prompt Flows: Pipelines combining models, data, moderation
  • Lifecycle Management: Development ‚Üí testing ‚Üí production

üí° Project Best Practices

  • Use Case Focus: Create projects per use case rather than per developer
  • Environment Separation: Keep environments separated (Chatbot Project, Summarization Project, etc.)
  • Responsible AI: Enable Responsible AI configuration at the project level
  • Resource Isolation: Ensure projects don't interfere with each other

üéØ Hands-on Practice

  1. Create a project inside your hub called "HR-Policy-Chatbot"
  2. Configure Responsible AI settings for the project
  3. Set up project-specific access controls

3. Deploy Necessary Services

Core Azure Services

When setting up a project, you typically provision core Azure services:

ü§ñ A. Azure OpenAI Service

  • Models: Provides GPT, embeddings, and DALL¬∑E models
  • Use Cases: Text generation, chat, summarization, code, and image creation
  • Requirements: Requires quota approval (not available in every subscription by default)
  • Deployment: Deploy in the same region as your other services for optimal performance

üîç B. Azure AI Search (Cognitive Search)

  • Functionality: Provides indexing + semantic search capabilities
  • RAG Essential: Essential for RAG pattern (Retrieval-Augmented Generation)
  • Data Enrichment: Can enrich data with OCR, entity recognition, and translation before indexing
  • Integration: Connects with storage accounts and AI services

üíæ C. Storage Accounts

  • Blob Storage: Store unstructured docs (PDF, images)
  • Database Storage: CosmosDB / SQL DB for structured datasets (products, customer info)
  • Knowledge Base: Used as the raw knowledge base before indexing in AI Search
  • Access Control: Configure appropriate access levels and encryption

üîê D. Azure Key Vault

  • Secret Management: Stores API keys, credentials, secrets
  • Security Best Practice: Recommended instead of embedding keys directly in code
  • Managed Identity: Can integrate with Managed Identity to remove need for key sharing
  • Access Control: Centralized access management and rotation
üõ°Ô∏è
Security & Access Considerations

üîí Security Best Practices

  • Least Privilege: Apply least privilege access at both hub and project levels
  • Managed Identity: Use Managed Identity for apps instead of API keys
  • Secret Storage: Store all secrets (OpenAI keys, connection strings) in Key Vault
  • Network Security: Enable network restrictions if needed (e.g., private endpoints for storage + search)

üîê Access Control Strategy

  • Hub Level: Define broad access patterns and governance policies
  • Project Level: Implement specific access controls for individual projects
  • Service Level: Configure service-specific permissions and restrictions
  • Monitoring: Track access patterns and audit security events

üéØ Exam Trick Questions & Common Pitfalls

‚ùì Question: What's the top-level organizational container in Azure AI Foundry?

Answer: Hub
Trick: Don't confuse with projects - hubs are the top-level containers that can contain multiple projects.

‚ùì Question: Which service is essential for RAG (Retrieval-Augmented Generation) patterns?

Answer: Azure AI Search
Trick: Don't choose Azure OpenAI alone - RAG requires both OpenAI for generation and AI Search for retrieval.

‚ùì Question: What's the best practice for storing API keys in Azure AI solutions?

Answer: Azure Key Vault with Managed Identity
Trick: Don't choose hard-coding or environment variables - Key Vault provides centralized, secure key management.

‚ùì Question: Which service requires quota approval before deployment?

Answer: Azure OpenAI Service
Trick: Don't assume all services are immediately available - OpenAI requires quota approval and isn't available in every subscription by default.

‚ùì Question: How should you organize projects within a hub?

Answer: By use case, not by developer
Trick: Don't create projects per developer - organize by business use case for better resource management and governance.

‚ùì Question: What's the recommended access control strategy for AI services?

Answer: Least privilege access at hub and project levels
Trick: Don't use broad access permissions - implement least privilege access for better security.

‚ùì Question: Which storage service is best for unstructured documents like PDFs?

Answer: Azure Blob Storage
Trick: Don't choose CosmosDB or SQL - these are for structured data, while Blob Storage is designed for unstructured files.

‚ùì Question: What's the first step in a typical AI deployment workflow?

Answer: Create a Hub
Trick: Don't start with services or projects - always begin with creating the top-level hub container.
