Responsible AI Principles

Core Principles

Microsoft requires all Azure AI deployments to follow Responsible AI (RAI) principles:

  • Fairness: Models should treat all users fairly. Avoid bias in loan approval models or facial recognition.
  • Reliability & Safety: Models must behave as expected in different conditions. Include fallback logic when AI confidence is low.
  • Privacy & Security: Protect user data (encryption, anonymization, least-privilege access). Ensure API keys are secured in Key Vault or use Managed Identity.
  • Inclusiveness: Ensure accessibility (multi-language, speech-to-text for hearing impaired).
  • Transparency: Users should know when they are interacting with AI. Provide explanations for decisions (responsible outputs).
  • Accountability: Human oversight is critical; AI must not make unchecked decisions.

Deployment Options

Real-time Endpoints

  • Used for interactive apps (chatbots, speech recognition)
  • Latency-sensitive, billed per request
  • Example: Deploy GPT-4, call via Python SDK

Batch Inference

  • Used for large document/image processing in bulk
  • Example: Summarizing 1M PDF documents overnight

Container Deployment

  • Supported by Vision, Speech, Document Intelligence
  • Run services in Docker containers at the edge or on-prem
  • Ideal for compliance-sensitive industries (healthcare, finance)

Default Endpoint

Endpoint Configuration

Every Azure AI resource has:

  • Endpoint URL: e.g., https://myai.openai.azure.com/
  • API Key: primary/secondary
  • Location: Found in Keys and Endpoints blade in Azure Portal

üí° Best Practice

  • Store keys in Key Vault
  • Use Managed Identity whenever possible
  • Always use the default endpoint for SDK/API calls ‚Üí keeps code portable

Container Deployment

Docker-based Deployment

Azure AI services (Vision, Speech, Document Intelligence) support Docker-based deployment:

  • Download from Microsoft Container Registry (MCR)
  • Run locally with docker run
  • Point container at your Azure subscription for billing

Advantages

  • Works offline (if pre-licensed)
  • Control over latency and data residency
  • Ideal for defense, healthcare, finance sectors

Choose Batch When:

  • Cost optimization needed

  • Data residency requirements
  • Offline operation needed
  • Compliance restrictions
  • Edge computing scenarios

üéØ Exam Trick Questions & Common Pitfalls

‚ùì Question: Which deployment method should you use for processing 10,000 invoices overnight?

Answer: Batch Inference
Trick: Don't choose real-time endpoints for large-scale processing - batch is more cost-effective and appropriate for non-interactive tasks.

‚ùì Question: For a healthcare application requiring data to stay on-premises, which deployment option?

Answer: Container Deployment
Trick: Only containers allow true on-premises deployment with data residency control. Real-time and batch are cloud-only.

‚ùì Question: What's the difference between Azure OpenAI and Azure AI Language for text analysis?

Answer: Azure AI Language is for pre-built text analysis (sentiment, NER), while Azure OpenAI is for generative tasks (chat, content creation).
Trick: Don't use expensive generative models for simple text analysis tasks.

‚ùì Question: Which service should you use for extracting structured data from forms?

Answer: Document Intelligence (Form Recognizer)
Trick: Don't confuse with Azure AI Vision OCR - Document Intelligence is specifically designed for structured data extraction from business documents.

‚ùì Question: For a chatbot that needs to answer questions about company policies, which architecture?

Answer: Azure AI Search + Azure OpenAI (RAG pattern)
Trick: RAG requires both semantic search (AI Search) and language generation (OpenAI). Don't pick just one service.

‚ùì Question: Which SDK should you use for sentiment analysis?

Answer: azure-ai-textanalytics
Trick: Don't use the OpenAI SDK for simple text analysis - use the specialized Azure AI Language SDK for better performance and cost.

‚ùì Question: What's the best practice for storing API keys?

Answer: Azure Key Vault or Managed Identity
Trick: Never store API keys in code or configuration files. Always use secure key management.

‚ùì Question: For a real-time speech translation app, which services are needed?

Answer: Azure AI Speech (Speech Translation)
Trick: This is NOT Azure AI Language translation - Speech handles audio-to-audio translation, Language handles text-to-text.
